# Transaction Types per Day

This dashboard tracks how individual **transaction type families** evolve over time, revealing daily patterns in protocol operations and revenue sources.

## Overview

<DailyTrendsDashboard />

## Understanding the Chart

The stacked area chart provides a time-series view of treasury revenue with the following characteristics:

### Chart Elements

- **X-axis**: Date (chronological)
- **Y-axis**: Daily revenue in SOL
- **Stacked Areas**: Each colored area represents a different category (token, type, or pool depending on grouping mode)
- **Total Height**: The top edge shows total daily revenue across all categories

### Grouping Modes

Switch between different perspectives using the controls:

- **By Token**: See how different token inflows contribute to daily revenue over time
- **By Type**: Track how transaction types (fees, liquidations, transfers, etc.) evolve
- **By Pool**: Monitor which liquidity pools drive daily activity

### What to Look For

- **Spikes and Anomalies**: Sudden increases often indicate large liquidations or protocol events
- **Compositional Shifts**: Changes in the relative height of areas show evolving revenue sources
- **Trends**: Upward or downward slopes reveal growth or decline patterns
- **Volatility**: Fluctuation patterns indicate revenue stability or variability

### Interactive Features

- **Click on areas**: Select a specific category to view its top transactions in the table below
- **Hover**: See detailed breakdown for any day including exact SOL amounts per category
- **Zoom**: Use the date range controls to focus on specific time periods

## Top Transactions Table

The interactive table below the chart displays:
- **Default view**: Top transactions across the entire date range
- **Filtered view**: When you click on a specific area (token/type/pool), see the top transactions for that category

This allows you to investigate the specific events behind daily revenue patterns.

## Use Cases

### Trend Analysis
- Identify growth periods and their drivers
- Spot seasonal or cyclical patterns
- Detect changes in protocol activity levels

### Composition Analysis
- See which revenue sources are gaining or losing importance
- Understand protocol diversification over time
- Track the impact of new features or pools

### Event Investigation
- Investigate revenue spikes by clicking to see top transactions
- Correlate revenue patterns with protocol or market events
- Validate expected outcomes from protocol changes

## Methodology

Daily revenue is calculated from the **realized types attribution system**:

1. Each day's transactions are classified and attributed
2. Revenue is grouped by the selected dimension (token, type, or pool)
3. Values are stacked to show composition while maintaining total visibility
4. All SOL equivalents are calculated using same-day conversion rates

The data maintains 100% attribution accuracy, ensuring the total shown matches actual on-chain treasury inflows.
