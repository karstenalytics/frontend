# Orca vs. Fusion

This dashboard compares the **cumulative revenue performance** of the two main liquidity pool ecosystems: **Orca Whirlpools** and **DefiTuna's Fusion AMM**.

## Understanding the Comparison

The cumulative revenue chart shows the stacked area comparison between Orca and Fusion pool ecosystems:

### Chart Elements

- **X-axis**: Date (chronological from protocol start)
- **Y-axis**: Cumulative revenue in SOL
- **Stacked Areas**: Each colored area represents accumulated revenue from different pools
- **Pool Filtering**: Use the grouping controls to focus on pool-level comparison

### Key Comparison Metrics

This visualization helps compare:

- **Total Contribution**: Which ecosystem has generated more cumulative revenue?
- **Growth Trajectory**: Which pools are growing faster over time?
  - Steeper slopes = faster revenue accumulation
  - Flatter slopes = slower or stable growth
- **Inflection Points**: When did each ecosystem's contribution rate change?
- **Relative Performance**: How does Orca revenue compare to Fusion at any point in time?

### Orca Whirlpools

**Orca** is a concentrated liquidity DEX on Solana with:
- **Established Market**: Long-standing protocol with deep liquidity
- **Wide Token Support**: Many trading pairs available
- **Trading Fees**: Revenue from swap fees on liquidity positions
- **Mature Infrastructure**: Battle-tested pools and integrations

### DefiTuna Fusion AMM

**Fusion** is DefiTuna's native automated market maker featuring:
- **Protocol Integration**: Direct integration with DefiTuna treasury
- **Custom Pools**: Pools designed for DefiTuna's specific strategies
- **Fusion-specific Features**: Unique AMM mechanisms
- **Strategic Control**: Direct control over pool parameters

### Grouping Modes

Switch between perspectives:

- **By Pool**: Compare individual Orca vs. Fusion pools (recommended for this analysis)
- **By Token**: See which tokens flow through each ecosystem
- **By Type**: Understand transaction types per ecosystem

### Interactive Features

- **Click on pool areas**: View top transactions for that specific pool
- **Hover**: See exact cumulative totals for Orca vs. Fusion at any date
- **Date range controls**: Focus on specific time periods to analyze performance shifts

## Methodology

Cumulative revenue is calculated by:

1. Starting from the protocol's first treasury transaction
2. Adding each day's realized revenue to the running total
3. Maintaining separate cumulative totals for each category (token/type/pool)
4. Stacking these totals to show both composition and total growth

All values maintain 100% attribution accuracy, ensuring the cumulative total matches the exact on-chain treasury inflows since inception.
